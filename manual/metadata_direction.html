Direction

A Direction(pushtopics__Direction__c) is a reusable component that defines the Source and Target Connections, i.e. a 'direction' of data transfer. The Source is the Connection where the data is retrieved from, whereas the Target is the Connection where the data is synced to. A Direction can be specified for a Pipeline, which consists of a number of object level Mappings which will utilize the same direction of data transfer. Alternatively, a Direction can be specified for a single Mapping, which would override the Direction specified in the overall Pipeline. The following table describes the relevant fields in detail.

Best Practices:

  • A good naming convention is always helpful. Usually, a Direction can be named as source => target or source2target etc.
  • Installing DSP in the source org and having the Current Org checked on the Connection record will gain the maximum parallelism and hence will have the best performance. The following depicts performance (data processing speed) from fastest to slowest: 1. Current => Current 2. Current => Integration, 3. Integration => Current, 4. Integration => Integration.
