Connection

A Connection(pushtopics__Connection__c) defines authorization settings that can be used to connect to a Salesforce instance, which can be used either as a Source where the data is retrieved from, or as a Target where the data is synced to.

Fields

NameApi NameRequiredDescription
API Namepushtopics__ApiName__cYThe API Name of a Connection. It is a unique and external Id field, by default hidden from the page layout and always defaulted to the Name field value.
Connection NameNameNThe Name of a Connection.
Is Current Org?pushtopics__IsCurrentOrg__cNIndicates whether the specified Connection is the current org or not. If checked, no other credential is needed, the data execution will run as the current user via APEX, instead of web services.
Is Sandbox?pushtopics__IsSandbox__cNUsed in conjunction with the field Username and Password + Security Token, this indicates whether the specified Connection is a sandbox or not.
Named Credentialpushtopics__NamedCredential__cNThe developer name (not the label) of the Named Credential that is used to connect with a Salesforce org. The required setting for the Named Credential:

Identity Type must be Named Principal, Authentication Protocol must be OAuth 2.0 or Password Authentication, Allow Merge Fields in HTTP Header and Allow Merge Fields in HTTP Body must be checked.Use https://login.salesforce.com for production and https://test.salesforce.com for sandbox as the URL when the protocol is Password Authentication.
Password + Security Tokenpushtopics__SalesforcePassword__cNThe Salesforce Password + Security Token used in conjunction with the Username and Is Sandbox? to authorize a Salesforce org.
Usernamepushtopics__Username__cNThe Username to use to connect to the specified org.
