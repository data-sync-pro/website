Batch Execution

A Batch Execution(pushtopics__BatchExecution__c) record has similar set of fields as an Execution. The difference is it logs the details at the batch level instead of the Execution level. A Batch Execution has a master-detail relationship with the Execution object.

Details

NameDeveloper NameDescription
Batch Execution NumberNameName of the Batch Execution, auto number.
Executionpushtopics__Execution__cMaster-detail relationship to the Execution object.
Original Batch Executionpushtopics__OriginalBatchExecution__cThe Batch Execution that this current Batch Excution was orginated from.
Actionpushtopics__Action__cFormula field, equals to the Action field on the Execution.
Start Timepushtopics__StartTime__cThe time an Execution started
End Timepushtopics__EndTime__cThe time an Execution ended
Succeeded?pushtopics__Succeeded__cIndicates whether the Batch Execution was succeeded or not.
Retrieved Countpushtopics__RetrievedCount__cThe records count retrieved from the source for the current batch.
Retrieved Data in Action Countpushtopics__RetrievedDataInActionCount__cThe count of retrieved records which were included in the action in the current batch. For example, if there were 200 records retrieved, of which 50 were new, and the Action was "Insert" only, this value would be 50.
Actioned Countpushtopics__ActionedCount__cThe count of records actioned against the target in the current batch. For example, if there were 100 insert and 50 update records retrieved from the source, and each update record had 2 matches in the target, this value would be 100 + 50*2 = 200, instead of 150.
Succeeded Countpushtopics__SucceededCount__cThe count of records that succeeded in the action against the target.
Failed Countpushtopics__FailedCount__cThe count of records that failed in the action against the target.

Execution Log

NameDeveloper NameDescription
Retrieve Query Stringpushtopics__RetrieveQueryString__cThe query string issued against the source.
Retrieved Data/Idspushtopics__RetrievedDataOrIds__cThe retrieved source data or IDs in JSON format.
Action Detailspushtopics__ActionDetails__cThe details of the action, including the source Id to target Id's map for the Update part of action, and source Id's for the Insert part of action.
Action Raw Responsespushtopics__ActionRawResponses__cThe raw response from the target as a result of an either Insert, Update or both Actions. If the target is connected via integration, the raw result also includes the status code and status, in addition to what is tracked when the target is the local org.
Exceptionspushtopics__Exceptions__cException message raised during the execution.
Resultspushtopics__Results__cStores a comprehensive view of the results in JSON. Each entry in the JSON will have the source data Id and target data Id for an Update action, "Success" which indicates success or fail, and "Errors" with the error details. If "Include Failed Data in Results?" is checked on the Mapping, the failed target data will be included as well. Copy and paste the JSON body to any JSON formatter/prettifier to better view well-structured data.
This field combines the necessary data points together to help with troubleshooting. And it does not comprise the permissions on data for the authorized user.
