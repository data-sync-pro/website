Fields Mapper

Fields Mapper is a component on the Mapping record page where the field level mappings or transformations from the source to the target can be configured.

Upon clicking the Fields Mapper tab, DSP fetches the target fields' metadata for the target object defined in the Mapping, and merges with the related Field Mappings for review (without saving). The field mapping entries are displayed in the following 4 category buckets in order, and within each bucket, mapped fields are shown prior to the un-mapped fields as sub-buckets. Within each sub-bucket, the fields are ordered alphabetically.

  • The target key field
  • The reference type of fields
  • The rest mappable fields
  • The non-mappable fields

Buttons

There are 3 buttons available in the component.

Clear Mappings

Clicking the button will clear all the existing fields' Mappings in the UI (without saving in the database).

Add Default Mappings

Clicking the button will map the source fields with the target fields automatically in the UI(without saving in the database) if the source object has the same field as the target field, in terms of the field's type and API name.

Save Mappings

Upon clicking the button, DSP saves all the mapped field entries as Field Mappings in the database, and deletes all the irrelevant Field Mappings if there are any such existing. The Field Mapping object has a master-detail relationship with the Mapping object.

Field Level Mapping

Viewable checkbox

Every mappable field can be checked as "Viewable", which tells the "View Source Data" component to display the source fields mentioned in the "Map with Source" field.
