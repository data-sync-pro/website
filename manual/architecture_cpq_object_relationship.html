DSP Object Descriptions

This table focuses on what each object does inside DSP.

Name Description
Connection A Connection represents a connected Salesforce org and specifies the authentication required to establish the connection.
Direction A Direction defines the source and target Connections used by Executables, serving as a reusable configuration for where data is retrieved and where actions are applied.
Pipeline A Pipeline groups and organizes related Executables—either for contextual organization or for defining a specific execution or display sequence. It enables ordered execution of Batch jobs and Triggers, and ordered display of Data Lists and Action Buttons. A Pipeline also supports shared configurations such as Retrieve Criteria and Scope Filter, and provides access to the Migration Builder for creating structured data-migration Executables across Connections, and more.
Executable An Executable is the core configuration unit of DSP’s rules engines. It defines the logic for processing, transforming, and acting on Salesforce data—and powers Batch, Triggers, Data List, Action Button, and Data Loader engines. Depending on whether it’s linked to a Direction or Connection, it can move data between objects, apply actions, or generate data lists.
Field Mapping A Field Mapping associated with an Executable defines the transformation logic for individual target fields. It leverages source fields, expressions, and predefined functions to shape data.
Variable A Variable is a named, reusable formula that can be referenced in mappings, filters, or other expressions across one or more Executables.
Schedule A Schedule specifies the frequency and timing for executing a job item—such as a Pipeline, Executable, or Apex class—at defined intervals.
Pipeline Execution A Pipeline Execution is a log record that tracks the batch execution of a Pipeline containing multiple Executables, capturing the overall status and outcome of those Executions.
Execution An Execution is a log record that tracks a single run of a Batch Executable, recording its status, timing, and key metrics.
Batch Execution A Batch Execution represents one chunk within an Execution. It captures status, scope, outcome, resource usage, and metrics for that subset of records.
Saved Query A Saved Query stores a reusable, user-defined query—enabling quick access, editing, and execution from Query Manager (Q).
