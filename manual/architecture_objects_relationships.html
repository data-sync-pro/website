DSP Object Definitions

The table below lists the core DSP configuration and log objects, showing each object’s API name, its parent relationships, and a concise description.

Name API Name Parent Relationships Description
Connection pushtopics__Connection__c A Connection represents a connected Salesforce org and specifies the authentication required to establish the connection.
Direction pushtopics__Direction__c Connection A Direction defines the source and target Connections used by Executables, serving as a reusable configuration for where data is retrieved and where actions are applied.
Pipeline pushtopics__Pipeline__c Direction

A Pipeline groups and organizes related Executables—either for contextual organization or for defining a specific execution or display sequence.

It enables ordered execution of Batch jobs and Triggers, and ordered display of Data Lists and Action Buttons.

A Pipeline also supports shared configurations such as Retrieve Criteria and Scope Filter, and provides access to the Migration Builder for creating structured data migration Executables across Connections, and more.

Executable pushtopics__Executable__c Direction, Connection, Pipeline

An Executable is the core configuration unit of DSP’s rules engines. It defines the logic for processing, transforming, and acting on Salesforce data—and powers all five engines: Batch, Triggers, Data List, Action Button, and Data Loader.

Executables can be associated with either a Direction or a Connection. When linked to a Direction, the Executable specifies the source and target objects, along with the processing logic between them. When linked to a Connection, it supports defining an action on a target object using input from a specified data profile, or creating a single data list based on a query.

Executables provide a modular, declarative, and simplified yet powerful configuration model for managing complex operations within Salesforce.

Field Mapping pushtopics__FieldMapping__c Executable A Field Mapping associated with an Executable defines the transformation logic for individual target fields. It leverages source fields, expressions, and a list of predefined functions to shape the data transformation.
Variable pushtopics__Variable__c Connection, Executable A Variable is a named, reusable formula that can be referenced in mappings, filters, or other expressions across one or more Executables.
Schedule pushtopics__Schedule__c Executable, Pipeline A Schedule specifies the frequency and timing for executing a job item—such as a Pipeline, Executable, or Apex class—at defined intervals.
Pipeline Execution pushtopics__PipelineExecution__c Pipeline A Pipeline Execution is a log record that tracks the batch execution of a Pipeline containing multiple Executables, capturing the overall status and outcome of the associated Executions.
Execution pushtopics__Execution__c Executable, Pipeline Execution An Execution is a log record that tracks the execution of a Batch Executable, capturing its status, timing, and key details.
Batch Execution pushtopics__BatchExecution__c Execution A Batch Execution is a log record representing the execution of a single batch chunk within an Execution. It captures the status, scope, outcome, resource usage, and other key details related to processing that specific subset of records.
Saved Query pushtopics__SavedQuery__c A Saved Query stores a reusable, user-defined query—enabling easy access, editing, and execution from Query Manager (Q).
