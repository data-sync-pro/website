There are 3 methods of authorizing a Salesforce org, choose one of the following three options:

Connection

Option #1: Current Org

Check Is Current Org? to indicate whether the current Connection is the current org or not. If checked, no other credential is needed, the data execution will run as the current user via APEX, instead of web services.

Option #2: Username + Password

Type in your Username and Security Token + Password, and check Is Sandbox? if the Connection is a sandbox, and click Save. On the saved record, click the Test Connection quick action to check if the credential is entered correctly.

Note:

Password + Security Token is an encrypted text field. Users with View Encrypted Data permission, and having the Object, record and field level access will be able to view the decrypted value.

Option #3: Named Credential

In general, using the Named Credential is one of the most secure methods of authorizing an external web service endpoint in Salesforce. This is because no packages or custom code can restore the crendentials defined in a Named Credential, which is protected by Salesforce at the platform level.

Create your Named Credential in the setup section and add its developer name into the Connection record you are creating. DSP supports 2 types of Named Credentials - OAuth2.0 and Username + Password. The required settings for the Named Credential are as following:

Identity Type must be Named Principal, Authentication Protocol must be OAuth 2.0 or Password Authentication, Allow Merge Fields in HTTP Header and Allow Merge Fields in HTTP Body must be checked. Use https://login.salesforce.com for production and https://test.salesforce.com for sandbox as the URL when the protocol is Password Authentication.

Note:

Remote Site Setting is a key security measure that prevents API callouts to unauthorized network addresses. A Remote Site Setting is required for Connections other than current org. DSP automatically creates the Remote Site Settings upon Connection creation or when a Connection is updated and the credential is changed, so long as the credential is valid, and the current user has the permission Customize Application or Modify All Data. If the current user does not have the permission to add a Remote Site Setting, check with your system administrator and add the Remote Site Setting accordingly. DSP never adds Remote Site Settings other than for the Connections authorized by your users.

There could be 3 Remote Site Settings created when a Connection is saved, if not already existing -

  • https://login.salesforce.com
  • https://test.salesforce.com
  • The custom domain name of the authorized org
